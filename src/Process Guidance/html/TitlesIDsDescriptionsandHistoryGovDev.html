Titles, IDs, Descriptions and History (GovDev)
Send Feedback

You use the Title and ID fields to uniquely identify work items in a list. You use the Description and History fields to provide additional information that is needed to implement the work item and to track changes that are made to the work item.

After a work item is created, you can modify each of these fields except for ID. When you create and save a work item, the ID is assigned by Team Foundation and cannot be changed.

The Title, ID, Description , and History fields are used to track information and changes for all work item types provided with the Microsoft Solutions Framework (MSF) for Agile Software Development - v5.0 process template.

The record of changes that were made to the work item after it was created. Every time that the work item is updated, information is appended to the history, which specifies the date of the change, who made the changes, and which fields were changed.
